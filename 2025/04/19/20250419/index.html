DFS

DFS 逐行分析

  • 定義一個圖（Graph），用「鄰接表（adjacency list）」的方式來表示。
  • 每個鍵（例如 'A'）代表一個節點，每個值是一個列表，代表與該節點相連的節點。
  • 範例中：
    • 'A' 連到 'B' 和 'C'
    • 'B' 連到 'D' 和 'E'
    • 'C' 連到 'F'
    • 'D', 'F' 為終點（無子節點）
  • 定義一個名為 dfs_recursive 的函式，使用 遞迴（recursive） 實作 DFS。
  • 參數：
    • graph: 圖的結構
    • node: 當前訪問的節點
    • visited: 一個用來記錄已訪問節點的集合（初始值為 None）
  • 第一次進入函式時會建立一個空集合 visited，記錄哪些節點已被訪問過，避免重複與無限遞迴。
  • 如果目前的節點還沒有被訪問，就進入處理邏輯。
  • 印出目前訪問的節點，表示這個節點是 DFS 的一部分。
  • 將該節點加入 visited 集合中，避免未來重複造訪。
  • 遍歷目前節點的所有「鄰接節點（neighbor）」
  • 對每一個相鄰節點再呼叫一次 dfs_recursive，這就是「遞迴」的核心！
  • 呼叫 dfs_recursive 函式，從節點 'A' 開始整個 DFS 搜尋流程。

BFS

BFS 逐行分析

  • 定義一個圖，使用 鄰接表（Adjacency List） 方式。
  • 節點 'A' 連到 'B' 和 'C'，這是一個有向圖。
  • 定義一個名為 bfs 的函式，接收：
    • graph：圖結構
    • start：起始節點
  • 建立一個空集合 visited，用來記錄已經拜訪過的節點。
  • 建立一個 list 來模擬 queue，並將起始節點 start 放進去。
  • 注意：list 是先進先出結構（FIFO），不過效率不如 deque。
  • 只要 queue 不是空的，就持續進行廣度優先搜尋（BFS）。
  • 從 list 的開頭取出第一個元素（模擬 queue 的出列動作）。
  • 注意：list.pop(0) 會導致後面元素整體移動，時間複雜度是 O(n)。
  • 如果這個節點尚未拜訪過，就繼續處理。
  • 如果已拜訪過，則跳過（避免重複或無限循環）。
  • 印出當前拜訪的節點，作為 BFS 的遍歷結果。
  • 把當前節點加入 visited 集合，標記為「已拜訪」。
  • 逐一檢查當前節點的所有鄰居（相鄰的節點）。
  • 如果鄰居還沒被拜訪，就將它加入 queue，等待後續處理。
  • 從節點 'A' 開始，呼叫 bfs 函式，進行廣度優先搜尋。
